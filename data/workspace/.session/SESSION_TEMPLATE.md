# Session Title
_简短的会话标题，5-10 个字，信息密度高_

# Current State
_当前正在做什么？有哪些待完成的任务？下一步是什么？_

# Task specification
_用户要求做什么？有哪些设计决策或上下文？_

# Files and Functions
_涉及哪些重要文件？它们的内容和作用是什么？_

# Workflow
_通常按什么顺序跑哪些命令？输出怎么解读？_

# Errors & Corrections
_遇到了什么错误？怎么修的？用户纠正了什么？哪些方法失败了不要再试？_

# Codebase and System Documentation
_系统的关键组件是什么？怎么协同工作？_

# Learnings
_什么方法有效？什么无效？要避免什么？不要和其他章节重复_

# Key results
_如果用户要求特定输出（答案、表格、文档），在这里重复最终结果_

# Worklog
_按时间顺序，尝试了什么、做了什么？每步简要记录_
